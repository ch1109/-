# 金融原型机首页交互设计优化方案

## 一、AI Agent产品交互模式调研总结

### 1.1 主流AI Agent产品交互特征

基于对ChatGPT、Claude、Perplexity等主流AI产品的分析,总结出以下核心交互模式:

#### **ChatGPT交互模式**
- **对话式交互**: 简洁的一问一答界面,对话历史按时间顺序排列
- **思考过程可视化**: 在复杂任务中显示"Thinking..."状态
- **流式输出**: 文字逐字显示,增强实时感
- **多模态支持**: 文本、图片、代码混合展示
- **上下文保持**: 自动记忆对话历史,支持多轮对话

#### **Claude交互模式**
- **Artifacts工作区**: 将AI生成的成果(代码、文档、图表)独立展示在右侧工作区
- **思考链展示**: 可展开查看AI的推理过程(Thinking blocks)
- **成果与对话分离**: 左侧对话,右侧成果,互不干扰
- **版本管理**: 支持对生成内容的迭代和版本对比
- **实时预览**: 代码和设计即时渲染

#### **Perplexity交互模式**
- **搜索+对话融合**: 将搜索结果整合到对话流中
- **来源引用**: 每个回答都标注信息来源
- **相关问题推荐**: 主动建议用户可能感兴趣的后续问题
- **知识图谱**: 可视化展示信息之间的关联
- **Pro模式**: 深度分析时显示多步骤推理过程

### 1.2 行业最佳实践总结

| 设计原则 | 具体实践 | 应用价值 |
|---------|---------|---------|
| **成果与过程分离** | Claude的Artifacts模式 | 避免对话区被大量内容淹没,用户可专注于成果 |
| **思考过程透明化** | 可折叠的Thinking blocks | 建立信任,让用户理解AI决策逻辑 |
| **渐进式信息披露** | 默认折叠详细信息,点击展开 | 降低认知负担,保持界面简洁 |
| **实时反馈** | 流式输出、进度指示器 | 减少等待焦虑,增强互动感 |
| **上下文感知** | 智能推荐后续问题 | 引导用户深入探索,降低使用门槛 |
| **多模态融合** | 文本+图表+数据卡片 | 适应不同信息类型,提升理解效率 |

### 2.2 Agent工作区设计原则

#### **核心理念: 展示成果,而非过程**

**✅ 应该展示的内容:**
- 数据可视化图表(饼图、柱状图、雷达图、趋势线)
- 结构化数据卡片(关键指标、对比数据)
- 方案选项卡片(可滑动、可点击)
- 进度指示器(步骤条、百分比)
- 交互式组件(滑块、选择器、按钮)
- 实时更新的画像雷达图

**❌ 禁止出现的内容:**
- "正在分析您的数据..." (过程性描述)
- "我认为您适合..." (AI主观表述)
- "根据我的判断..." (解释性语言)
- 大段文字说明
- AI的思考日志

#### **设计规范:**

1. **卡片式布局**
   - 每个信息模块独立成卡片
   - 卡片间距: 16px
   - 圆角: 12px
   - 阴影: 0 2px 8px rgba(0,0,0,0.08)

2. **色彩系统**
   - 主色调: #1890FF (品牌蓝)
   - 成功色: #52C41A (绿色)
   - 警告色: #FAAD14 (橙色)
   - 危险色: #F5222D (红色)
   - 背景色: #F5F7FA (浅灰)
   - 卡片背景: #FFFFFF

3. **动态更新机制**
   - 数据变化时使用过渡动画(300ms ease-in-out)
   - 新增内容从上方滑入
   - 图表数据更新使用数值动画
   - 进度条平滑推进

   - 发送按钮始终可见

## 三、状态转换流程设计

### 3.1 三种核心状态定义

| 状态 | 名称 | 主要特征 | 工作区内容 | 聊天区内容 |
|-----|------|---------|-----------|-----------|
| **状态1** | 待机态 | 等待用户交互 | 功能卡片首页 | AI能力介绍+建议问题 |
| **状态2** | AI工作态 | AI主导服务 | 动态工作台(图表/方案) | 完整对话+思考过程 |
| **状态3** | 业务辅助态 | 传统业务流程 | 业务办理界面 | 步骤提示+实时帮助 |

### 3.2 状态切换触发条件

```mermaid
graph TD
    A[状态1: 待机态] -->|用户说"我想理财"| B[状态2: AI工作态]
    A -->|用户点击"账户管理"| C[状态3: 业务辅助态]
    B -->|用户说"帮我办理开户"| C
    C -->|用户说"给我推荐理财方案"| B
    B -->|用户点击"返回首页"| A
    C -->|用户点击"返回首页"| A
```

## 五、设计亮点与创新

### 5.1 核心创新点

1. **双区域协同联动**
   - 聊天区的对话触发工作区的内容更新
   - 工作区的操作同步到聊天区记录
   - 两个区域互不干扰但信息一致

2. **思考过程透明化**
   - 用户可选择查看AI的推理过程
   - 建立信任,消除"黑盒"疑虑
   - 教育用户理解AI决策逻辑

3. **渐进式信息披露**
   - 默认只显示核心信息
   - 详细内容折叠,按需展开
   - 避免信息过载

4. **实时动态反馈**
   - 画像雷达图随回答实时"生长"
   - 进度条平滑推进
   - 数据变化有动画过渡

5. **上下文智能推荐**
   - 根据当前状态推荐后续问题
   - 预判用户可能的疑虑
   - 主动提供帮助

### 5.2 用户体验优势

| 传统设计 | 优化后设计 | 体验提升 |
|---------|-----------|---------|
| 所有信息混在对话中 | 成果独立展示在工作区 | 信息层次清晰,易于理解 |
| AI思考过程不可见 | 可选择查看思考链 | 建立信任,消除疑虑 |
| 等待时无反馈 | 实时进度+Agent协同日志 | 降低焦虑,增强参与感 |
| 被动等待AI提问 | 智能推荐后续问题 | 降低使用门槛,提升效率 |
| 静态数据展示 | 动态图表+实时更新 | 增强互动感,提升吸引力 |

## 六、技术实现建议

### 6.1 前端技术栈
- **框架**: Vue 3 + TypeScript
- **UI组件库**: Element Plus / Ant Design Vue
- **图表库**: ECharts / Chart.js
- **动画库**: GSAP / Anime.js
- **状态管理**: Pinia
- **路由**: Vue Router

### 6.2 关键技术点

1. **双区域布局**: CSS Grid / Flexbox
2. **实时通信**: WebSocket / Server-Sent Events
3. **流式输出**: 逐字显示动画
4. **图表动画**: ECharts动画配置
5. **状态管理**: 全局状态机管理三种状态

### 6.3 性能优化

- 虚拟滚动(聊天历史过长时)
- 图表懒加载
- 组件按需加载
- 防抖节流(输入框、滚动)

## 十一、总结

### 11.1 核心创新点

本方案的核心创新在于:

1. **双区域协同设计**: 将AI工作成果与对话过程分离,既保持信息清晰,又提供完整上下文

2. **思考过程透明化**: 让用户可选择查看AI的推理过程,建立信任,消除"黑盒"疑虑

3. **状态智能切换**: 根据用户意图自动切换工作模式,无缝衔接AI服务与传统业务

4. **主动式帮助**: AI不是被动等待,而是主动检测用户困难并提供帮助

5. **渐进式信息披露**: 默认简洁,按需展开,避免信息过载

### 11.2 与行业最佳实践的对标

| 设计元素 | 对标产品 | 本方案实现 |
|---------|---------|-----------|
| 成果与对话分离 | Claude Artifacts | Agent工作区独立展示成果 |
| 思考过程可视化 | ChatGPT Thinking | 可折叠的思考链模块 |
| 实时协同展示 | Perplexity Pro | Agent协同日志实时更新 |
| 智能推荐 | 所有主流AI | 上下文感知的问题推荐 |
| 流式输出 | 所有主流AI | 逐字显示+动画效果 |
