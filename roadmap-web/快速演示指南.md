# 金融原型机 - 快速演示指南

## 🎯 演示目标

向用户展示优化后的AI交互体验,重点突出:
1. **更AI化**: AI的智能理解和主动服务能力
2. **更自然**: 人格化、情感化的对话体验
3. **动态UI生成**: AI根据用户需求实时生成界面和内容

---

## 🚀 快速开始

### 1. 启动项目

```bash
cd roadmap-web
npm run dev
```

访问: http://localhost:5175

## 📋 演示流程A: AI工作模式 (教育金规划)

### 演示时长: 5-7分钟

### 演示脚本

#### 第1步: 进入AI工作模式 (30秒)

**操作:**
- 点击首页的"待机态 → AI工作模式"按钮

**讲解要点:**
```
大家看,这是我们的待机态首页。现在我点击"AI工作模式"按钮,
模拟用户说"我想给孩子存教育金"。
```

**观察重点:**
- ✅ 界面分为左右两区: Agent工作区(60%) + 聊天交互区(40%)
- ✅ AI主动欢迎,介绍自己是"小智"
- ✅ 使用emoji和亲切的语言

---

#### 第2步: 展示AI思考过程 (1分钟)

**操作:**
- 点击第一条AI消息下方的"💭 查看AI思考过程"

**讲解要点:**
```
这是我们的核心创新点之一。用户可以看到AI的完整思考过程,
包括意图识别、知识检索、策略选择和风险检查。

注意这里有具体的数据:
- 置信度98%
- 已匹配872只基金
- 采用"苏格拉底式提问法"

这些细节让用户感受到AI的专业性和可信度。
```

**观察重点:**
- ✅ 思考过程结构化展示
- ✅ 有具体数据支撑
- ✅ 说明策略选择的原因

---

#### 第3步: 展示Agent协同日志 (1分钟)

**操作:**
- 点击"⚙️ 查看Agent协同日志"

**讲解要点:**
```
这里展示了4个Agent的协同工作过程:
- 意图识别引擎 (0.3秒)
- 数据分析师 (0.5秒)
- 财富规划师 (0.8秒)
- 市场研究员 (1.0秒)

每个Agent都有明确的工作内容和成果,让用户看到
"有一个AI团队在为我服务",而不是单一的AI。

这体现了我们的多Agent架构优势。
```

**观察重点:**
- ✅ 多Agent协同工作
- ✅ 每个Agent有具体成果
- ✅ 有时间维度,体现效率

---

#### 第4步: 渐进式提问 (2分钟)

**操作:**
- 依次回答5个问题:
  1. 孩子年龄: 选择 5岁
  2. 目标金额: 选择 50万
  3. 家庭月收入: 选择 2万
  4. 月固定支出: 选择 1万
  5. 现有储蓄: 选择 10万

**讲解要点 (问题1):**
```
注意每个问题都有详细的描述,解释"为什么要问这个问题"。

比如第一个问题,AI解释说:
"这将帮助我计算可用的投资时间。距离上大学的时间越长,
投资策略就越灵活。"

这种解释性的对话,让用户理解AI的意图,而不是机械地填表。
```

**讲解要点 (问题2):**
```
第二个问题提供了参考数据:
- 国内本科约30-50万
- 国内本科+研究生约50-80万
- 海外留学约100-150万

这帮助用户做决策,体现AI的专业性和实用性。
```

**讲解要点 (问题3):**
```
第三个问题涉及隐私,注意AI的表达:
"方便透露一下您家庭的月收入大概是多少吗?"

使用"方便透露一下"而不是直接问,更礼貌。
同时强调"这个信息会严格保密,仅用于评估您的储蓄能力"。

点开思考过程,可以看到AI采用了"隐私保护策略",
强调数据加密传输,符合个人信息保护法。
```

**观察重点:**
- ✅ 每个问题都有描述和解释
- ✅ 提供参考数据帮助决策
- ✅ 隐私问题特别处理
- ✅ 左侧雷达图实时更新

---

#### 第5步: 动态UI生成 (1分钟)

**操作:**
- 观察左侧Agent工作区的变化

**讲解要点:**
```
这是我们的另一个核心创新点: AI根据用户需求动态生成UI。

注意左侧的变化:
1. 用户画像雷达图 - 随着回答实时增长
2. 可行性评估卡片 - 自动计算储蓄率
3. 投资方案对比 - 生成两个方案供选择

这些UI都是AI根据用户的具体情况动态生成的,
不是预设的模板。

比如储蓄率是根据用户的收入和支出实时计算的,
投资方案也是根据用户的风险偏好和时间跨度定制的。
```

**观察重点:**
- ✅ 雷达图动态增长
- ✅ 储蓄率实时计算
- ✅ 方案动态生成
- ✅ 进度条动画

---

#### 第6步: 总结亮点 (30秒)

**讲解要点:**
```
总结一下流程A的核心亮点:

1. 透明化AI思考过程 - 建立信任
2. 多Agent协同工作 - 体现专业性
3. 渐进式提问策略 - 降低用户压力
4. 动态UI生成 - 个性化体验
5. 隐私保护强调 - 增强安全感

这些设计让用户感受到:
"这不是一个冷冰冰的机器人,而是一个专业、可信、
有温度的AI理财顾问团队。"
```

## 📋 演示流程B: 业务辅助模式 (风险测评)

### 演示时长: 4-5分钟

### 演示脚本

#### 第1步: 进入业务辅助模式 (30秒)

**操作:**
- 返回首页,点击"待机态 → 业务辅助模式"按钮

**讲解要点:**
```
现在演示第二个流程: 业务辅助模式。

这个模式模拟的是传统银行业务办理场景,
但加入了AI的智能辅助。

注意界面布局:
- 左侧(60%): 传统业务办理区
- 右侧(40%): AI聊天辅助区
- 顶部: 5步进度条,始终可见
```

**观察重点:**
- ✅ 5步进度条清晰可见
- ✅ AI主动欢迎并介绍流程
- ✅ 告知总步骤和预计时间

---

#### 第2步: AI陪伴式服务 (1分钟)

**操作:**
- 点击"开始核验"按钮

**讲解要点:**
```
注意AI的欢迎消息:

"您好!我是AI小智,很高兴为您服务。😊
这个流程一共有5个步骤,大约需要3-5分钟。"

AI明确告知:
1. 总共几步 - 设定预期
2. 需要多长时间 - 降低焦虑
3. 具体怎么操作 - 提供指引

这种"陪伴式服务"让用户感觉不是一个人在办理业务,
而是有一个AI助手在旁边随时帮助。
```

**观察重点:**
- ✅ AI主动介绍流程
- ✅ 提供具体操作指引
- ✅ 强调"随时可以问我"

---

#### 第3步: 步骤完成反馈 (30秒)

**操作:**
- 等待身份核验完成(自动完成)

**讲解要点:**
```
注意AI的反馈:

"太好了!身份核验完成 ✓"

使用"太好了!"这种情感化表达,给予用户正向激励。

然后详细介绍下一步:
- 为什么要做风险测评 (监管要求)
- 有多少道题 (5道题)
- 需要多长时间 (2分钟)
- 没有对错之分 (降低压力)

这种详细的引导,让用户清楚知道接下来要做什么。
```

**观察重点:**
- ✅ 情感化反馈
- ✅ 详细的下一步引导
- ✅ 进度条更新到第2步

---

#### 第4步: AI主动介入 ⭐核心亮点 (1.5分钟)

**操作:**
- 开始答题,前4题快速选择
- 第5题"流动性需求"停留30秒不选择

**讲解要点:**
```
这是流程B的核心创新点: AI主动介入。

AI检测到用户在第5题停留了30秒,判断用户可能遇到困难,
主动弹出黄色帮助提示框。

注意AI的帮助内容:

1. 同理心表达:
   "是不是对这个概念不太熟悉?没关系,让我用大白话给您解释一下!"

2. 概念解释:
   用最简单的语言解释"流动性需求"

3. 场景化举例:
   🏠 高流动性 - 明年买房准备首付
   👨‍👩‍👧 中流动性 - 3年后孩子上大学
   👴 低流动性 - 10年后退休养老

4. 产品建议:
   每个场景都给出具体的产品建议

这种主动介入+场景化帮助,大大降低了用户的理解难度,
提升了完成率。
```

**观察重点:**
- ✅ AI主动检测用户困惑
- ✅ 黄色提示框醒目
- ✅ 场景化举例易于理解
- ✅ 提供具体产品建议

---

#### 第5步: 测评结果展示 (1分钟)

**操作:**
- 完成第5题,查看测评结果

**讲解要点:**
```
注意测评结果的展示方式:

1. 庆祝表达:
   "🎉 恭喜!风险测评完成!"

2. 结构化结果:
   - 总分: 12分
   - 风险等级: R3 (稳健型)
   - 评估结论: 适合中等风险产品

3. 适合产品列表:
   ✓ 混合基金
   ✓ 债券基金
   ✓ 结构性存款

4. 结果用途说明:
   "根据监管要求,我们只能向您推荐与您风险等级匹配的产品,
   这是为了保护您的资金安全。"

5. 有效期说明:
   "测评结果将保存在您的账户中,有效期1年。"

这种详细的结果展示,让用户清楚知道:
- 我的结果是什么
- 我能买什么产品
- 这个结果有什么用
- 结果保存多久
```

**观察重点:**
- ✅ 情感化庆祝
- ✅ 结果详细清晰
- ✅ 列出具体产品
- ✅ 解释结果用途

---

#### 第6步: 总结亮点 (30秒)

**讲解要点:**
```
总结一下流程B的核心亮点:

1. 陪伴式服务 - AI全程陪伴,随时解答
2. 进度可视化 - 5步进度条,清晰明了
3. AI主动介入 - 检测困惑,主动帮助
4. 场景化举例 - 用生活场景解释专业术语
5. 详细反馈 - 每一步都有清晰的反馈和引导

这些设计让传统的业务办理流程变得:
"不再枯燥,不再困惑,有AI陪伴,轻松完成。"
```

## 💡 演示技巧

### 1. 节奏控制
- 不要太快,给观众时间观察界面变化
- 重点功能要停留讲解,不要一带而过
- 适时暂停,询问观众是否有疑问

### 2. 重点突出
- 用鼠标圈出重点内容
- 重复强调核心创新点
- 对比优化前后的差异

### 3. 互动引导
- 邀请观众提问
- 询问观众的第一感受
- 收集改进建议

### 4. 故事化表达
- 用"假如我是用户"的视角讲解
- 描述用户的心理变化
- 强调用户的痛点和解决方案
